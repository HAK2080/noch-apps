## Bugs Found and Fixed

### BUG-1: Login Race Condition — Navigate Before State Commits

**Severity:** Critical (users couldn't log in reliably)

**Root cause:** `Login.jsx` called `navigate('/')` immediately after `await signIn()`, before React had committed the updated `user` state to the context. `ProtectedRoute` saw `user === null` and redirected back to `/login`, creating a redirect loop.

**Fix (two-part):**

1. `src/contexts/AuthContext.jsx` — `signIn()` now eagerly sets `user` and loads `profile` before returning:
   ```js
   if (data.user) {
     setUser(data.user)
     await loadProfile(data.user.id)
   }
   ```

2. `src/pages/Login.jsx` — navigation moved from `handleSubmit` into a `useEffect` triggered by `user` state change:
   ```js
   useEffect(() => {
     if (user) navigate('/', { replace: true })
   }, [user])
   ```

---

### BUG-2: Infinite Redirect Loop at Root (`/`)

**Severity:** High (cascades from BUG-1; any session with slow profile load looped forever)

**Root cause:** `RootRedirect` in `App.jsx` redirected to `/login` when `profile === null`, even when `user` was authenticated. After login, `onAuthStateChange` briefly sets `loading=false` and `profile=null` while the profile row is being fetched. This sent authenticated users to `/login`, which immediately redirected them back to `/`, creating an infinite loop.

**Fix:** Added `user` to `RootRedirect`'s destructuring and distinguished the two states:
```js
// Before:
if (!profile) return <Navigate to="/login" replace />

// After:
if (!user) return <Navigate to="/login" replace />   // not authenticated
if (!profile) return null                              // authenticated but profile loading — wait
```

---

### BUG-3 (Discovery): React StrictMode Breaks Playwright Auth Against Dev Server

**Severity:** Medium (test infrastructure only; not a production bug)

**Root cause:** React 18 StrictMode double-mounts effects in development, causing `getSession()` and `onAuthStateChange` to fire twice with non-deterministic ordering. Playwright tests against `http://localhost:5173` saw `navigate → /login → /login` loop patterns and intermittent auth failures.

**Fix:** Playwright configured to test against the production build (`vite preview --port 4173`) which does not use StrictMode's double-effect behavior.
